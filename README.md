 B’s Therapy 🌸🐝 — Website Project Documentation 

## 1. Introduction

B’s Therapy is a professional beauty brand specialising in eyelash extension services. The purpose of this website is to establish an online presence, promote services, and provide a platform for potential clients to view work and make enquiries.

The website has been designed to reflect a modern, luxury beauty aesthetic while maintaining usability and accessibility. It combines both visual appeal and functional interactivity to create an engaging user experience.

---

## 2. Target Audience Analysis

The primary target audience includes:

* Females aged 16–35
* Individuals interested in beauty, self-care, and aesthetics
* Clients seeking professional lash technicians

### User Needs:

* View examples of previous work (portfolio/gallery)
* Easily navigate services and pricing
* Contact or book appointments quickly
* Trust the brand through professional presentation

### How the Website Meets These Needs:

* High-quality real images build authenticity and trust
* Clean layout ensures easy navigation
* Mobile responsiveness supports on-the-go users
* Interactive features improve engagement

---

## 3. Features and Functionality

### 3.1 Navigation System

The website includes a responsive navigation bar with links to key sections such as Home, About, Services, and Contact.

* A **hamburger menu** is implemented for mobile devices
* A **sticky navigation bar** improves usability during scrolling
* Active page highlighting helps users identify their location

---

### 3.2 Image Gallery (Portfolio)

A gallery section is used to showcase real lash extension work completed by B’s Therapy.

* Grid-based layout for visual organisation
* Hover effects for interactivity
* **Lazy loading** improves performance and loading speed
* Real images increase credibility and professionalism

---

### 3.3 Slideshow Component

A JavaScript-powered slideshow is included on the homepage:

* Automatically cycles through featured images
* Manual controls allow user interaction
* Enhances visual engagement and highlights key content

---

### 3.4 Form Validation and User Feedback

The website includes both a contact form and newsletter subscription form.

* Input validation ensures correct data entry (e.g., valid email format)
* Error handling provides immediate feedback
* Toast notifications improve user experience by displaying real-time messages

---

### 3.5 Additional Interactive Features

* **Scroll-to-top button** improves navigation
* **Smooth scrolling** enhances user experience
* **Sticky navbar with shadow effect** provides visual feedback
* **Lazy loading images** optimise performance

---

## 4. Technologies Used

### HTML 

Used to create the semantic structure of the website, including sections, forms, and navigation.

### CSS 

Used for styling, layout, and responsiveness:

* Grid and Flexbox for layout
* Hover effects and transitions
* Colour scheme and typography

### JavaScript 

Used to implement interactivity:

* Slideshow functionality
* Form validation
* Toast notifications
* DOM manipulation
* Event handling

---

## 5. Design and Aesthetic Choices

### Colour Scheme

Soft pink tones and neutral colours were chosen to reflect femininity, beauty, and luxury. This aligns with industry standards in beauty branding.

### Typography

Simple, clean fonts were used to ensure readability and professionalism.

### Layout

A minimalist layout was chosen to:

* Avoid clutter
* Highlight images
* Improve user navigation

### Imagery

Real client images were used instead of stock images to:

* Build trust
* Showcase actual skills
* Create authenticity

---

## 6. User Experience (UX) and Accessibility

The website was designed with usability as a priority:

* Responsive design ensures compatibility across devices
* Clear navigation reduces user confusion
* Visual hierarchy guides user attention
* Feedback systems (toasts) improve interaction

### Accessibility Considerations:

* Alt text added to images
* Simple navigation structure
* Readable font sizes and contrast

---

## 7. Testing and Evaluation

### Testing Methods:

* Tested on different screen sizes (mobile, tablet, desktop)
* Checked all links and navigation
* Validated forms with both correct and incorrect inputs
* Ensured JavaScript features function correctly
